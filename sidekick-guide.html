GPI Sidekick guide

Please note that sheet to doc and doc to sheet sync are created to better work in a team.

GPI Sidekick for sheets guide

1. Sync with doc

To sync with doc, you should have at least two columns.

  • First Column with the heading of ID.
  • Second column containing some data with some heading.
  • Then at the end, we have "attached_single_doc" and/or "attached_doc" column. They represent attached doc for the full sheet or attached doc for every row respectively. If they do not exist already, you can create them.

Sync limitations in case of attached_single_doc:

  • If we add a new row in the sheet, it will not be synced to the existing attached_single_doc document. For that, we have to delete the existing link and initialize again to create a new document.
  • We cannot change the ID of the document once the document is created because it is synced based on the row ID present in the document and the sheet.

GPI Sidekick For Docs Guide

1. Sync with sheet

To sync with the sheet, you should have at least two tables.

  • First table containing data of a single row in the sheet. It should have the heading representing the row in the sheet.
  • The heading of the data represents the ID row name in the sheet.
  • Second table containing the metadata of the sheet. It contains a spreadsheet link and sheet name. It should have a heading of 'Metadata'.

Also remember that sheet data needs to be in a specific format. Please refer to the 'GPI Sidekick for sheets guide' section below to see that guide.
